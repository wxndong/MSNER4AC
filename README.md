# 介绍
本项目提出了一个简单的多策略古汉语命名实体识别系统，在EvaHan比赛中评估结果中得到了F1值 (Test Total) 84.91 的成绩，大于baseline F1值 (Test Total) 80.61

# 说明
- 模型设计解释：Task B选择Softmax而不是CRF是因为不想对trainsetB做额外处理，其序列长度平均值是29左右，因此直接用Softmax训练更快，原因是：时间复杂度层面CRF 是 $O(n^2)$ > Softmax $O(n)$，相比于训练代价成倍增长，精读有些损失是可以接受的
- 习率设置可能有问题：没有搜索学习率，所以学习率不是5e-5就是3e-5
- 由于版权问题，GitHub仓库暂不上传数据集及结果
- 可读性问题：因为用到了CRF这个库，其不允许非法标签-1存在
  - BERT的PAD部分使用0，并且使用Attention Mask在各阶段确保计算合法性
  - 为确保Attention Mask 正确应用，重写了部分Trainer类的方法
